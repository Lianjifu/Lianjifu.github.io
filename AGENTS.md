# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Project Overview

Personal blog site at [lianjifu.cn](https://lianjifu.cn) — a **pre-built static site** generated by [Hexo 5.4.0](https://hexo.io/) with the [NexT 7.8.0](https://theme-next.js.org/) theme (Pisces scheme), deployed to GitHub Pages.

**This repo contains the generated output (`hexo generate`), not the Hexo source files.** The source Markdown posts and Hexo config (`_config.yml`, `package.json`, `source/_posts/`) live in a separate source repo.

## Tech Stack

- **Static site generator**: Hexo 5.4.0
- **Theme**: NexT 7.8.0 (Pisces scheme)
- **Deployment**: GitHub Pages + custom domain (CNAME → lianjifu.cn)
- **Search**: Algolia + local search (`search.xml`)
- **Animations**: Velocity.js, Pace.js loading bar
- **Widget**: Live2D (hibiki model)
- **Icons**: Font Awesome 5
- **Content**: Chinese-language tech blog posts (Python tutorials)

## Key Commands

```bash
# Serve locally (requires Hexo source repo)
hexo server

# Generate static files (requires Hexo source repo)
hexo generate

# Deploy to GitHub Pages (requires Hexo source repo)
hexo deploy

# Clean generated files (requires Hexo source repo)
hexo clean
```

## Workflow

1. Write posts as Markdown in the Hexo source repo (`source/_posts/`)
2. Run `hexo generate` to produce the static output (this repo)
3. The static output is deployed to GitHub Pages via `hexo deploy` or manual push

This repository is the **deployment target** — most editing will happen in the Hexo source repo rather than here. Modifications to `css/`, `js/`, `lib/`, theme config, or `source/` content in the source repo require re-generation.
