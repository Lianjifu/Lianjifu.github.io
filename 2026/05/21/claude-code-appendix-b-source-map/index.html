附录 B：源码导航

思考维度 内容
引导思考 展开
它为什么存在
  • 1800 个 TypeScript 文件找起来像大海捞针——功能分类索引比文件名排序管用多了
  • 帮读者按功能区域定位关键文件，而不是盲目 grep
它解决什么问题
  • Claude Code 的目录按功能域组织但不扁平——部分工具散落在 tools/、services/tools/、utils/ 里
  • 读完了某一章可以秒定位到实现文件——不用在 1800 个文件里翻
它在系统中的位置
  • 作为附录不参与主线：但看完每一章都可能需要回来查文件路径
  • 9 个分类覆盖全书 25 章涉及的源码区域
它如何工作
  • 每个条目包含文件路径、大小估计、职责描述和相关章节
  • 按功能模块分组：入口与启动 → 核心引擎 → 工具系统 → Agent → 权限 → MCP → 状态 → UI → 基础设施
它如何实现
  • 纯 Markdown 表格，路径相对于 src/
  • 行数标记"大小估计"而非精确行数——因为 DCE 后产物大小和源码行数不是一回事
不同平台如何做
  • React 官方文档：用目录树文本图，强调目录结构而非功能索引
  • Monorepo 工具（Nx）：通过依赖图可视化，和本书的功能分类互补
优势是什么？
  • 优势
    • 功能分类索引比目录树更贴近读者需求
    • 80+ 条记录标注大小和职责，方便评估代码量
它解决什么问题
  • Claude Code 的目录结构按功能域组织但不扁平——部分工具分散在 tools/、services/tools/、utils/ 等目录
  • 读者读完某一章后可以快速定位到对应的实现文件
它在系统中的位置
  • 作为附录不参与主线叙事
  • 9 个分类模块覆盖了全书 25 章涉及的所有源码区域
它如何工作
  • 每个文件条目包含路径、大小估计、职责说明和相关章节
  • 按功能模块分组：入口点与启动 → 核心引擎 → 工具系统 → Agent 系统 → 权限安全 → MCP → 状态管理 → UI → 工程基础设施
它如何实现
  • 纯 Markdown 表格，路径相对于 src/ 目录
  • 大小估计以源码行数计，标注"实际产物体积因 DCE 和 minification 缩小"
不同平台如何做
  • 传统源码文档（如 React、Vue）：通常使用目录树文本图或源码层级遍历，强调目录结构而非功能索引
  • Monorepo 工具（Nx、Turborepo）：通过依赖图可视化项目结构，与本章的分类索引互补
优缺点是什么
  • 优势
    • 功能分类索引比目录树更贴近读者的阅读需求
    • 80+ 条记录覆盖源码主体，标注大小和职责方便评估代码量

本附录提供 Claude Code 源码库的功能分类索引，帮助读者快速定位关键源码文件。所有路径相对于 src/ 目录。源码总量约 1800 个 TypeScript/TSX 文件，合计约 49 万行代码。

思考笔记

入口点和启动流程是理解系统的"门"——所有功能都从这里开始。

  • cli.tsx 是二进制实际入口，main.tsx 是逻辑入口——路径分流保证快速响应。
  • init() 被 memoize 保证全局只执行一次。
  • feature() 编译时 DCE 让不同构建产物体积差异巨大。
  • 启动时间线精确到每毫秒——profileCheckpoint 提供了测量基础。

思考笔记

QueryEngine 是系统的心脏——所有对话都通过它流转。

  • submitMessage 是唯一入口——SDK 和 REPL 共用同一抽象。
  • mutableMessages 贯穿生命周期，状态在多次调用间保持。
  • 查询循环是 while(true) + break 条件——状态机驱动而非异常驱动。
  • 错误恢复链（413 → collapse → compact → fallback）覆盖极端场景。

思考笔记

工具系统是 Claude Code 的能力边界——40+ 内置工具通过统一接口集成。

  • Tool<Input, Output, Progress> 三参数泛型约束输入、输出、进度。
  • buildTool 工厂 + TOOL_DEFAULTS 实现 fail-closed 安全默认。
  • 并发调度器（StreamingToolExecutor）管理工具的并行执行和结果保序。
  • 三层防线（单次截断 + 磁盘持久化 + 全局预算）防止工具输出撑爆上下文。

思考笔记

权限和安全是生产级 Agent 系统的基石——不是"加个认证"，而是"层层防御"。

  • 六层权限模式覆盖从完全锁定到完全信任的完整光谱。
  • Bash 安全的 18 个文件、23 个安全检查器实现了纵深防御。
  • 规则优先级 deny > allow > ask——防火墙式的规则引擎。
  • 推测分类器通过并行安全检查和用户操作来隐藏验证延迟。

思考笔记

思考笔记

React + Ink 让 340+ 组件在终端中运行——终端不再是 UI 的荒漠。

  • Ink 通过 Yoga + WASM 实现 Flexbox 布局——终端中的 CSS。
  • 差异化输出（diff output）计算帧间差异，只输出变化——减少终端 I/O。
  • REPL 5000 行是"万法归宗"的入口——所有子系统的输出汇集于此。
  • Vim 模式状态机提供媲美编辑器的键盘交互体验。
